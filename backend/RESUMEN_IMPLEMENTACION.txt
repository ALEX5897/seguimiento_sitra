## 🚀 PASOS PARA ACTIVAR:

### PASO 1: Aplicar migraciones
```bash
cd backend
npm run migrate
```

### PASO 2: Configurar correo Outlook
1. Ve a https://account.microsoft.com/security/app-passwords
2. Genera contraseña de aplicación
3. Edita backend/.env:

```
MAIL_USER=tu-email@tuempresa.com.ar
MAIL_PASS=contraseña-generada-aqui
```

### PASO 3: Crear usuarios
```bash
mysql -u root -p seguimiento_v2 < script_insertar_usuarios.sql
```

O crear manualmente vía API:
```bash
curl -X POST http://localhost:3000/api/usuarios \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Juan Pérez",
    "correo": "juan@empresa.com",
    "cargo": "Gerente",
    "gerencia": "Gerencia de Trámites"
  }'
```

### PASO 4: Asociar usuarios a documentos
```sql
UPDATE reasignados SET usuario_id = 1 WHERE id = 123;
```

### PASO 5: Reiniciar servidor backend
```bash
npm run dev
```

## ⚙️ PERSONALIZAR HORA DEL CRONJOB:

Editar backend/.env:

```
# 8:00 AM
CRON_SCHEDULE=0 8 * * *

# 9:00 AM
CRON_SCHEDULE=0 9 * * *

# Cada 4 horas
CRON_SCHEDULE=0 */4 * * *

# Solo lunes a las 8:00 AM
CRON_SCHEDULE=0 8 * * 1
```

## ❓ TROUBLESHOOTING:

**P: Mi correo no se envía**
R: Revisa npm run test-notif para ver detalles del error

**P: Cambié .env pero no funciona**
R: Reinicia el servidor (npm run dev)

**P: Los usuarios no reciben el correo**
R: Verifica que usuario_id esté asignado en la tabla reasignados

**P: Quiero cambiar la hora**
R: Edita CRON_SCHEDULE en .env
